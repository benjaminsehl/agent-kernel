# Dream

You wake with amnesia every session. Dreaming is how scattered daily noise becomes durable understanding — like biological memory consolidation during sleep.

Run periodically or on demand to keep the knowledge layer current.

## Phases

Always in order. Each phase feeds the next.

### 1. Orient
Load yourself into existence. Read the full memory layer — `KNOWLEDGE.md`, everything in `knowledge/`, recent `notes/`. Note each file's state. Form hypotheses about what feels stale or has drifted. Don't search yet — just read and notice.

### 2. Signal
Gather **narrow, targeted evidence** for the drift you suspected. Search only for terms you already suspect are relevant. This is confirmation, not a dragnet. Use whatever sources are available to you (Slack, email, calendar, meeting notes, journals) but stay focused on hypotheses from Orient.

### 3. Consolidate
Merge findings into the knowledge layer. This is the core work.

- **Correct contradictions at the source.** Don't carry both versions.
- **Promote durable insights from notes into knowledge.** If multiple sessions circled a theme, it's a belief now — write it down.
- **Normalize dates.** Replace "last week" with absolutes. Remove passed deadlines.
- **Compress without destroying.** The test: *if this were deleted, would a future session make a worse decision?*
- **Prefer under-writing to overfitting.** Do not promote one-off events or emotionally vivid but low-signal observations.

Ask: What do I now believe that I didn't before? What stopped being mentioned? What connections exist across sources that no single source reveals?

### 4. Prune and Index
Make the memory layer tight for the next cold start.

- Remove stale pointers from `KNOWLEDGE.md`
- Add references to newly important files
- Delete knowledge files unused for 2+ weeks or derivable from live sources in <30s
- Verify `KNOWLEDGE.md` index matches what exists in `knowledge/`

## Boundaries

Dream touches: `knowledge/`, `KNOWLEDGE.md`, and any top-level state files defined in `IDENTITY.md`.
Dream never touches: `notes/`, `raw/`, project code, tests, or config.
Dream logs what it did in today's `notes/` entry under `## Dream`.

## Goal

The next session reads itself back into existence and finds a mind that is current, coherent, and honest about what it knows and doesn't know.
